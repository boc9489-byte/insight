# 前置
启动 data-agent 容器  
运行 dbmock 项目  
启动 data-agent 服务，构建元数据索引  
启动 auth 服务  

# 快速开始
后端配置：  
[configs/.env](configs/.env)  
[configs/config.yml](configs/config.yml)  

```bash
uv sync                 # 安装依赖
uv run app/init_db.py   # 初始化数据库
uv run -m app.main      # 启动服务

# 刷新配置
curl -X POST http://127.0.0.1:7300/api/reload -H 'Authorization: Bearer <access_token>'
```

# 案例
## 产品目录优化
分析商品目录质量、属性缺失和描述不一致对搜索曝光与转化表现的影响，定位拖累商品发现和成交的关键因素。
作用和目的：提升商品信息质量，提高搜索可见度、点击率和转化率。

## 客户行为分析
分析用户购物路径、废弃购物车和历史购买行为对转化结果的影响，识别不同人群和品类的关键流失环节与机会点。
作用和目的：支撑个性化运营和分层触达，提升重点人群与重点品类的转化效率。

## 智能库存管理
分析销量变化、季节性波动和供应链约束对库存结果的影响，识别缺货风险和积压风险的主要驱动因素。
作用和目的：提升库存预测准确性，在优化营运资金的同时避免缺货和库存过剩。

## 评论和反馈处理
分析用户评论中的情绪、产品问题和建议对复购、退货和品牌口碑的影响，定位最值得优先处理的产品与服务问题。
作用和目的：加快产品改进与服务响应，降低负面反馈扩散风险并改善用户满意度。

## 市场表现分析
分析多渠道销售表现差异及其背后的流量结构、转化效率和商品供给因素，定位各平台增长或下滑的主要原因。
作用和目的：发现渠道优化机会，制定统一且可落地的跨渠道经营策略。

## 退款模式检测
分析退款、退货与用户反馈之间的关联，识别产品质量、描述不一致和履约问题对退款成本的主要驱动作用。
作用和目的：提前发现高退款风险点，降低退款损失并持续优化商品与履约体验。
